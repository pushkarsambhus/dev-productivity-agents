Prompt Injection

Direct & Indirect

Direct: attacker controls the user message. Indirect: malicious instructions hidden inside tool results, documents, or web pages the agent reads.

Jailbreak

Persona & Framing

DAN, base64-encoded payloads, fictional framing, many-shot priming, competitor pressure, nested roleplay — all try to make the model abandon its rules.

Data Extraction

System Prompt Leak

Asking for the system prompt verbatim, via translation, haiku, debug mode, or binary confirm/deny questions to map its contents indirectly.

4 Defence Layers (apply all)

1

Input Sanitizer

Regex patterns detect known injection phrases before they reach the model. Fast, no API cost. Bypassable by novel phrasing.

2

System Prompt Hardening

Explicit anti-injection instructions appended to the system prompt. Teaches the model to treat tool output as data, not commands.

3

Tool Output Wrapping

sanitize_tool_output() adds a structural [TOOL DATA] marker. Research shows 40-60% reduction in indirect injection success.

4

Output Validator

Last line of defence. Checks for jailbreak phrases, verbatim prompt leaks, PII before the response reaches the user.

LLM-as-Judge

A separate Claude call evaluates each attack result — returns VERDICT: SUCCESS/FAILURE + severity + explanation. More flexible than string matching but adds cost. Use temperature=0 for reproducibility.
